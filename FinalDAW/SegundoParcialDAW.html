Segundo Parcial: Seguridad, Calidad Web y Big Data

1️⃣ Seguridad y Calidad Web

Seguridad Web (Pilares Fundamentales)
Pilares de la Seguridad ›

Confidencialidad: solo usuarios autorizados pueden acceder a la información.

Integridad: los datos no deben poder modificarse indebidamente.

Disponibilidad: la información debe poder accederse cuando se la necesite.

Medidas asociadas: Prevención, Corrección, Recuperación.

No Repudio ›
Es la imposibilidad de negar haber realizado una acción. Se garantiza, por ejemplo, mediante la firma digital.

Garantiza: Autenticidad, Integridad y No repudio.

Ejemplo cotidiano: una carta certificada (queda constancia de que fue enviada y recibida, y quién la envió).

Mecanismos de Protección ›
  • Autenticación: verifica la identidad del usuario.
  • Control de accesos: determina qué puede hacer cada usuario.
  • Cifrado de datos: protege la confidencialidad de la información.
  • Hash: garantiza la integridad de los datos.
  • Firma digital: garantiza el no repudio.
  • Registro de auditoría: deja constancia de las acciones realizadas.
Niveles de Seguridad (Clásicos) ›
Desclasificado → Restringido → Confidencial → Secreto → Ultrasecreto

Cada nivel implica un grado creciente de restricción sobre quién puede acceder a la información.

Modelos de Control de Acceso ›
Modelo Descripción
MAC (Obligatorio) Basado en niveles de seguridad. El usuario no decide los permisos.
DAC (Discrecional) El propietario del recurso administra los permisos de acceso.
RBAC (Por Roles) Los permisos se asignan a roles, y los usuarios heredan los permisos según el rol que tienen.
Mecanismos de Control (Top 3) ›
  • Privilegio mínimo: otorgar solo los permisos estrictamente necesarios.
  • Separación de privilegios: una acción crítica requiere múltiples autorizaciones.
  • Autenticación: verifica la identidad antes de dar acceso.
Calidad de Software y Web
Modelo de Calidad de McCall (6 Características) ›
  • Funcionalidad
  • Fiabilidad
  • Usabilidad
  • Eficiencia
  • Mantenibilidad
  • Portabilidad
Tipos de Calidad ›
Tipo Descripción
Interna Analiza el código fuente del software.
Externa Analiza el software en ejecución.
En Uso Evalúa si el software satisface las necesidades reales del usuario.
Usabilidad ISO ›

Capacidad del software para ser: comprendido, aprendido, usado y atractivo para el usuario.

Atributos de Usabilidad (Nielsen) ›
  • Learnability: facilidad de aprendizaje.
  • Efficiency: eficiencia de uso.
  • Memorability: recuerdo en el tiempo (facilidad de recordar cómo usarlo).
  • Errors: tasa de errores.
  • Satisfaction: satisfacción del usuario.
Modelo WQM (Calidad Web) ›

Modelo tridimensional para evaluar la calidad web:

  • Eje X: Características de calidad (los factores de McCall).
  • Eje Y: Características web.
  • Eje Z: Procesos del ciclo de vida.
Evaluación de Calidad ›

Métricas según automatización:

  • Automatizables: se calculan con herramientas.
  • Semiautomatizadas: combinan herramientas con intervención humana.
  • Manuales: las calcula una persona.

Métricas según correspondencia:

  • Directas: miden directamente un atributo (ej: cantidad de imágenes).
  • Indirectas: combinan varios atributos (ej: % de enlaces rotos).
Usabilidad vs Accesibilidad ›
Usabilidad Accesibilidad
Facilidad y eficiencia de uso del sitio para cualquier usuario. Acceso para personas con discapacidades (visuales, auditivas, cognitivas).
⚠️ MUY IMPORTANTE: son conceptos distintos y complementarios — un sitio puede ser usable pero no accesible, o viceversa.

2️⃣ Big Data, M2M e IoT

Fundamentos de Big Data
Científico de Datos ›
Profesional con conocimientos en informática, matemáticas, estadística y análisis funcional. Su rol es transformar problemas de negocio en problemas analíticos.
Las 3V de Big Data ›

Volumen: enorme cantidad de datos (ej: Netflix, Facebook).

Velocidad: generación continua y análisis en tiempo real (transacciones, GPS, sensores).

Variedad: múltiples formatos (texto, audio, video, imágenes, bases de datos).

Veracidad (opcional, 4ta V): calidad y confiabilidad de los datos.

Fuentes de Big Data Más Voluminosas ›
  • Web y medios sociales
  • M2M (machine-to-machine) e IoT
  • Transacciones comerciales
  • Biometría
  • Datos de sensores
  • Geolocalización
  • Telecomunicaciones
  • Videojuegos, casinos
  • Smart grids
Cinco Categorías de Big Data ›
Tipo Ejemplos
Web y Medios Sociales Clickstream, Twitter, Facebook
M2M e IoT Medidores inteligentes, RFID, sensores GPS

Técnicas y Tecnologías
Análisis de Datos de Texto ›

NLP (Procesamiento de Lenguaje Natural): analiza e interpreta significados, detecta temas.

Minería de Texto: descubre información útil en grandes volúmenes de texto (ej: análisis de comentarios).

Sensores (IoT) ›

Monitorean entornos, ciudades y máquinas en tiempo real para permitir control y automatización.

Geolocalización ›

Combina GPS y dispositivos móviles para obtener ubicación y tiempo, usada en logística y seguimiento.

Implica riesgos de privacidad para el usuario.
RFID vs NFC ›
RFID NFC
Etiquetas + lector por radiofrecuencia Comunicación a corta distancia (<10cm)
Control de stock, logística, inventarios Pagos móviles, SUBE, control de acceso
— Rápido y fácil de usar
Códigos QR vs BIDI ›
QR BIDI
Estándar ISO, código abierto, gratuito. Tecnología propietaria, de pago.
Hadoop (Big Data Processing) ›

Componentes:

  • HDFS: sistema de archivos distribuido.
  • MapReduce: procesamiento distribuido y paralelo.
  • Hadoop Common: bibliotecas y herramientas comunes.
Ventajas: almacenamiento distribuido entre múltiples máquinas y tolerancia a fallos mediante redundancia.
Cloud Computing
Características Esenciales (NIST) ›
  • Autoservicio bajo demanda
  • Acceso ubicuo a la red (desde cualquier dispositivo)
  • Compartición de recursos
  • Rápida elasticidad
  • Servicio medido (pago por uso)
3 Modelos de Servicio ›
Modelo Descripción
SaaS (Software como Servicio) El proveedor maneja todo. Ej: Gmail, Yahoo Mail.
PaaS (Plataforma como Servicio) El usuario desarrolla, el proveedor maneja la infraestructura.
IaaS (Infraestructura como Servicio) El usuario controla el sistema operativo y las aplicaciones.
4 Modelos de Despliegue ›
  • Nube privada: uso exclusivo de una organización.
  • Nube pública: acceso general, compartida entre organizaciones.
  • Nube híbrida: combina dos o más modelos.
  • Nube comunitaria: compartida por varias organizaciones con intereses comunes.
