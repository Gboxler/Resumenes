Primer Parcial: Estado, Arquitecturas, Objetos ASP, CORBA, Cookies y HTML

1️⃣ Fundamentos y Manejo de Estado

HTTP es un protocolo Stateless (sin estado): cada petición es independiente y el servidor no recuerda nada de las anteriores por sí solo. Para simular que "recuerda" al usuario se usan mecanismos de manejo de estado como Session, Cookies y Application.
Mecanismos de Estado
Variables de Sesión (Session) ›

Persisten datos entre páginas, pero únicamente para esa sesión específica de ese usuario.

  • Alcance: Privado, cada usuario tiene su propia Session.
  • Duración: Mientras dure la sesión del usuario (hasta que expire o se cierre el navegador).
Cookies ›

Archivos de texto (máximo 4KB) que almacenan información para el intercambio entre cliente y servidor.

Tipo Descripción
No Consistentes Sin propiedad Expires. Mueren al cerrar el navegador (viven en RAM).
Consistentes Con Expires definido. Se guardan en el disco rígido y persisten entre sesiones.
Application ›

Datos compartidos globales entre todos los usuarios de la aplicación (no son privados como Session).

Como varios usuarios pueden acceder al mismo dato al mismo tiempo, requiere Lock y Unlock para evitar conflictos de concurrencia.

Archivos clave relacionados:

  • Web.config: Configuración específica de la aplicación web.
  • Machine.config: Configuración a nivel de servidor/máquina.
  • Global.asax: Maneja eventos globales de la aplicación (inicio, fin, sesión, errores).
XML y Estructuras Jerárquicas
XML ›

Lenguaje de marcado case-sensitive (distingue mayúsculas de minúsculas) diseñado para transportar y almacenar datos, no para mostrarlos visualmente.

PCDATA vs CDATA ›
PCDATA CDATA
Datos procesados (Parsed Character Data) que el parser interpreta y que forman la jerarquía del documento. Bloques que el parser ignora, usados para incluir caracteres especiales sin que se interpreten como marcado.
XSD - Cardinalidad ›

Un XSD (XML Schema Definition) valida la estructura de un XML. La cardinalidad indica cuántas veces puede aparecer un elemento.

Atributo Significado
minOccurs Cantidad mínima de veces que debe aparecer el elemento.
maxOccurs Cantidad máxima de veces que puede aparecer. El valor unbounded significa infinito (sin límite).

2️⃣ Arquitecturas y Modelos de Objetos

Objetos ASP (Cliente-Servidor)
REQUEST ›

Representa la entrada de datos que llegan del cliente al servidor.

  • Form: Datos enviados por un formulario con método POST.
  • QueryString: Datos enviados en la URL (método GET).
  • Cookies: Datos enviados dentro de las cookies del cliente.

Métodos de envío: POST / GET

RESPONSE ›

Representa la salida de datos del servidor hacia el cliente.

Métodos:

  • Write: Escribe contenido en la respuesta.
  • Redirect: Redirige al cliente a otra página.
  • Clear: Limpia el buffer de salida.
  • End: Finaliza el procesamiento y envía la respuesta.

Propiedad: Expires

SESSION ›

Almacena datos privados para cada usuario individual.

  • SessionID: Identificador único de la sesión del usuario.
  • Abandon: Finaliza la sesión actual.
  • Timeout: Tiempo de inactividad antes de que expire la sesión.
APPLICATION ›

Almacena datos globales, compartidos por todos los usuarios de la aplicación.

  • Lock / Unlock: Evitan que dos usuarios modifiquen el mismo dato al mismo tiempo.
  • OnStart: Evento disparado al iniciar la aplicación.
  • OnEnd: Evento disparado al finalizar la aplicación.
SERVER ›

Provee herramientas y utilidades del servidor.

  • CreateObject: Crea instancias de objetos/componentes en el servidor.
  • MapPath: Traduce una ruta virtual a una ruta física del servidor.
  • ScriptTimeout: Tiempo máximo de ejecución permitido para un script.
Sistemas Distribuidos - CORBA
CORBA ›
CORBA (Common Object Request Broker Architecture) permite la comunicación entre objetos escritos en distintos lenguajes de programación y ejecutados en distintas plataformas.
ORB (Object Request Broker) ›

Es el intermediario que gestiona la comunicación entre el objeto cliente y el objeto servidor, sin que cada uno necesite conocer los detalles técnicos del otro.

Stub y Skeleton ›

Son los "traductores" locales que permiten que cliente y servidor se comuniquen a través del ORB.

Stub Skeleton
Del lado del cliente: representa localmente al objeto remoto. Del lado del servidor: recibe la llamada y la reenvía al objeto real.
IDL (Interface Description Language) ›

Lenguaje utilizado para definir interfaces neutras, independientes del lenguaje de programación, que describen qué operaciones expone un objeto distribuido.

Compiladores IDL ›

Compiladores como Visibroker y Orbix generan automáticamente los Stubs y Skeletons a partir de la definición IDL, evitando escribirlos a mano.

DOM y Ciclo de Vida
Jerarquía del DOM ›
Window → Document → Form → Control → Propiedades

Cada nivel contiene al siguiente: la ventana contiene el documento, el documento contiene los formularios, los formularios contienen controles, y cada control expone sus propiedades.

Algoritmo del Servidor ›
Recepción → Procesamiento → Devolución

El servidor recibe la petición del cliente, la procesa (ejecuta lógica, consulta datos) y devuelve una respuesta.

HTML4 vs HTML5
Comparativa HTML4 vs HTML5 ›
Aspecto HTML4 HTML5
Semántica Etiquetas genéricas como <div> Etiquetas con significado: <header>, <footer>, <nav>
Multimedia Requiere plugins externos (Flash) Soporte nativo: <audio>, <video>
Almacenamiento Cookies (máx. 4KB) Web Storage (5-10MB), reemplaza a las cookies para guardar datos localmente
Gráficos Requiere herramientas externas <canvas> para gráficos 2D/3D nativos

3️⃣ Cookies, HTML Básico y Evaluación de Sitios

Cookies: Lectura y Escritura
Escritura de Cookies ›

Escritura simple:

Response.Cookies("Pais") = "Argentina"

Diccionarios (Subclaves): una misma cookie puede guardar varias subclaves.

Response.Cookies("color")("Fondo") = "Verde"
Lectura y Recursividad ›

Lectura de una subclave:

Request.Cookies("color")("Marcas")

Recorrido recursivo de subclaves: se verifica primero si la cookie tiene subclaves antes de recorrerlas.

If HasKeys Then For Each clave... ' recorrer subclaves Next End If
Cookies: Expiración
No Consistente vs Consistente ›
No Consistente Consistente
Sin .Expires → muere al cerrar el navegador (se guarda en RAM). Con .Expires definido → persiste en el disco rígido.
HTML Básico
Navegación ›
  • Dentro de la misma página: <a href="#id"> (ancla)
  • Hacia otra página: <a href="url">
Listas ›
Ordenadas Desordenadas
<ol> — atributos type ("1"/"A"/"I") y start (ej. start="15") <ul> — atributo type ("disc"/"circle"/"square")
Detalles Técnicos ›
  • Acentos: directos (con charset UTF-8) o mediante entidades (&aacute;, &eacute;, &ntilde;)
  • Sangrado: automático del navegador o controlado con CSS (margin-left, padding-left)
  • Comentarios: <!-- comentario -->
Conexión HTML + JavaScript
Vinculación e Interacción Básica ›

HTML: se enlaza el script externo dentro del <head>.

<script src="archivo.js"></script>

JavaScript: las funciones se definen en el archivo .js externo.

Entrada/salida de datos:

  • prompt(): solicita un dato de entrada al usuario.
  • eval(): convierte un texto en un valor numérico/evaluable.
  • alert(): muestra un mensaje de salida.
Evaluación de Sitios Web
Criterios de Evaluación ›
  • Tiempo de carga: debe ser razonable, no debe tardar mucho.
  • Coherencia visual: colores, fuentes y disposición consistentes en todo el sitio.
  • Propósito claro: debe entenderse de inmediato qué ofrece el sitio.
  • Enlaces alternativos: navegación interna, anclas, buscadores.
  • Regla de 3 clics: la información debe ser accesible sin necesitar muchos clics.
  • Usabilidad: interfaz intuitiva para el usuario.
  • Accesibilidad: debe incluir alt en imágenes, etiquetas semánticas, y evitar elementos obsoletos como <blink>.
Idea clave: un buen sitio web se evalúa tanto por su rendimiento técnico (carga, accesibilidad) como por su diseño (coherencia, usabilidad, propósito claro).
