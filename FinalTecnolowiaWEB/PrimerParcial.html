1️⃣ ¿Qué es una Aplicación Web?

Una aplicación web es un conjunto de funcionalidades que permite a los usuarios realizar tareas a través de un navegador. A lo largo del tiempo, la web ha evolucionado, pasando de ser un sistema estático con solo texto (Web 1.0) a permitir la interacción con el usuario (Web 2.0). En la Web 3.0, que es semántica, se da un papel importante a la inteligencia artificial y la personalización.
Evolución de la Web ›
Versión Descripción
Web 1.0 Información estática expresada en texto, sin interacción.
Web 2.0 Permite la interacción entre el usuario y la información, como en blogs, foros y redes sociales.
Web 3.0 Conocida como la web semántica, integra inteligencia artificial y personalización.
El Metaverso ›

El metaverso es una realidad virtual inmersiva, donde los usuarios interactúan mediante avatares. Características como la interconexión e ininterrupción permiten una experiencia más integrada entre el mundo físico y digital.

La Web 3.0 también está relacionada con la seguridad a través de blockchain.
Funcionalidad de las Aplicaciones Web ›

Las aplicaciones web transformaron la web de un entorno estático a uno interactivo. Estas aplicaciones están alojadas en servidores y permiten la interacción con el usuario, trabajando con un front end (interfaz) y un back end (servidor).

La evolución de herramientas como PHP y JavaScript facilitó la creación de aplicaciones dinámicas como Google y Facebook.

Arquitectura Cliente-Servidor ›

Los clientes solicitan servicios a los servidores.

Cliente Ligero Cliente Rico
El servidor gestiona los datos. El cliente gestiona la lógica de la aplicación.
El Desarrollador Front End ›

Trabaja con herramientas como HTML, CSS y JavaScript para crear la interfaz de usuario, utilizando frameworks como React y Angular.

La usabilidad y accesibilidad son claves para una buena experiencia de usuario, y el control de versiones con Git es esencial para gestionar el código y colaborar en proyectos.
Diferencia entre Aplicación Web y Página Web ›
Página Web Aplicación Web
Es un documento estático que muestra información al usuario, como texto e imágenes. No permite interacción compleja, solo visualización. Es un programa interactivo accesible a través de un navegador, que permite a los usuarios realizar tareas como llenar formularios, hacer compras o gestionar cuentas.

2️⃣ Tecnologías Cliente/Servidor, Servicios Web y APIs

Tecnologías del Lado del Cliente
HTML, CSS y JavaScript ›

HTML: lenguaje de marcado utilizado para crear páginas web y aplicaciones. Sus elementos forman la estructura de la página, representando texto, imágenes, formularios, entre otros.

CSS: se usa para controlar el diseño y la apariencia de los elementos HTML, permitiendo la personalización de la presentación visual de una página web.

JavaScript: lenguaje de programación dinámico e interpretado, utilizado para añadir interactividad a las páginas web. Se puede incluir dentro de elementos HTML o como archivos externos.

Frameworks y Librerías del Cliente ›

Angular (SPA de Angular): framework de desarrollo de aplicaciones web, mantenido por Google, que facilita la creación de aplicaciones de una sola página (SPA) y aplicaciones web progresivas (PWA).

React: biblioteca centrada en la creación de interfaces de usuario. Permite integrar HTML directamente dentro de su código JavaScript.

Vue: framework para crear interfaces de usuario interactivas. Se utiliza en aplicaciones de una sola página y puede combinarse con herramientas y bibliotecas adicionales para crear aplicaciones sofisticadas.

Blazor WebAssembly: permite la creación de aplicaciones web interactivas del lado cliente utilizando .NET y WebAssembly. Funciona en todos los navegadores modernos y permite la interoperabilidad con JavaScript.

Tecnologías del Lado del Servidor
JSP, Python, PHP y ASP.NET ›

JSP (Java Server Pages): lenguaje desarrollado por SUN Microsystems, que combina XML y scriptless para la creación de páginas dinámicas en Java.

Python: lenguaje de código abierto con diversas bibliotecas para el desarrollo web, utilizado en aplicaciones y sitios dinámicos.

PHP: lenguaje de programación que se incrusta dentro de HTML y genera contenido dinámico. Es ampliamente utilizado en plataformas como Wikipedia y Facebook.

ASP.NET: entorno de desarrollo de aplicaciones web que soporta múltiples lenguajes como VB, C# y Python, y se integra bien con bases de datos y servicios en la nube como SQL Server y Azure.

Servicios Web y APIs
Servicios Web ›

Los servicios web permiten la comunicación entre sistemas a través de la web. Utilizan protocolos como SOAP y XML, permitiendo ejecutar métodos de un sistema de manera remota, como consultar tasas de cambio en un servicio externo.

Cómo funcionan: cuando un sistema invoca un método remoto, un proxy se encarga de enviar la solicitud al servidor, que la procesa y devuelve el resultado en un formato compatible.

XML: formato de texto que presenta datos de manera estructurada y jerárquica, usado en servicios web para transferir información.

APIs ›
Una API es una interfaz que permite que las aplicaciones interactúen entre sí, facilitando el intercambio de datos y funcionalidades de manera segura.

Cómo funciona: una aplicación cliente hace una solicitud a la API, que a su vez consulta un servidor para obtener la información. La respuesta se envía de vuelta a la aplicación que hizo la solicitud.

Diferencia entre Web Services y APIs: los servicios web facilitan la interacción entre computadoras a través de la red, mientras que las APIs son interfaces que permiten la comunicación entre aplicaciones.

Consideraciones para el Desarrollo de Aplicaciones Web ›
  • Conocer las necesidades del negocio: es esencial entender los requisitos del usuario y el dominio del negocio para crear aplicaciones efectivas.
  • Interoperabilidad: determinar si el sistema debe integrarse con otros es crucial durante el análisis.
  • Elección del framework: se debe seleccionar el marco de trabajo según los objetivos del proyecto y las tecnologías necesarias (HTML5, CSS3, JavaScript, etc.).
  • Usabilidad y accesibilidad: el diseño debe ser sencillo, claro y adaptable a diferentes dispositivos.

3️⃣ Historia y Evolución de HTML

HTML (Hypertext Markup Language) es el lenguaje de marcado fundamental para la creación y estructuración de contenido en la web. A lo largo de su evolución, HTML ha sido clave en la expansión de la World Wide Web.
Orígenes ›

En 1989, Tim Berners-Lee, un científico británico, propuso un sistema para gestionar información en el CERN (Organización Europea para la Investigación Nuclear).

En 1990, junto con Robert Cailliau, Berners-Lee desarrolló un modelo de sistema de hipertexto para compartir información mediante una red.

En 1991, Berners-Lee introdujo el primer lenguaje de marcado: HTML, para estructurar y enlazar documentos.

Evolución de HTML ›
Año / Versión Hito
1994 Berners-Lee fundó el W3C (World Wide Web Consortium) para estandarizar tecnologías web, incluido HTML.
HTML 2.0 (1995) Primera versión estandarizada de HTML.
HTML 3.0 (1996) Introdujo nuevas características como tablas y formularios.
HTML 3.2 (1997) Agregó soporte para hojas de estilo (CSS) y mejoras en formularios.
HTML 4.0 (1997) Mejoró la semántica, permitió el uso de JavaScript y separó contenido de presentación.
XHTML (2000) Una versión más estricta basada en XML.
HTML5 (2008) Introdujo nuevos elementos semánticos, soportó multimedia sin complementos externos y mejoró la interoperabilidad.
HTML Living Standard A partir de HTML5, HTML pasó a un estándar "vivo", lo que implica actualizaciones continuas sin versiones específicas.
Front-End ›

Es la interfaz gráfica y la parte visible de una aplicación. Incluye lo que los usuarios ven e interactúan, y es responsable de mostrar la información de manera atractiva.

Responsabilidades:

  • Interfaz de Usuario (UI): diseño y presentación.
  • Interactividad: implementación de botones, formularios y menús.
  • Navegación: gestión intuitiva de la estructura de la aplicación.
  • Optimización del rendimiento: asegurar una carga rápida de recursos.

Tecnologías: HTML (estructura), CSS (diseño), JavaScript (interactividad) y frameworks como React, Angular y Vue.js para interfaces dinámicas.

Back-End ›

Es el lado del servidor que gestiona la lógica, el procesamiento de datos y la interacción con la base de datos.

Responsabilidades:

  • Lógica de Negocio: procesos internos según las necesidades del negocio.
  • Acceso a Base de Datos: almacenamiento y recuperación de datos.
  • Autenticación y Autorización: seguridad del sistema.
  • Integración de Servicios Externos: conexión con otras aplicaciones.
Etiquetas en HTML ›

Las etiquetas HTML son elementos clave para estructurar y dar formato al contenido en una página web. Son interpretadas por el navegador para mostrar la página correctamente.

Estructura de una etiqueta:

  • Se compone de un nombre de etiqueta rodeado de corchetes angulares.
  • Etiquetas de apertura y cierre: muchas etiquetas requieren una etiqueta de apertura y una de cierre.
  • Etiquetas de autocierre: algunas etiquetas no requieren una etiqueta de cierre.

Etiquetas HTML comunes:

Etiqueta Descripción
<html>Define el comienzo y final del documento HTML.
<head>Contiene metainformación sobre el documento.
<title>Define el título de la página, visible en la barra del navegador.
<body>Contiene el contenido principal de la página.
<h1> - <h6>Definen encabezados de diferentes niveles.
<p>Define un párrafo.
<a>Define un enlace (hipervínculo).
<img>Define una imagen.
<ul>, <ol>, <li>Definen listas ordenadas y no ordenadas.
<div>Agrupa elementos para aplicar estilos.
<span>Aplica estilos a un fragmento específico de texto.
Atributos en HTML ›

Los atributos en HTML proporcionan información adicional sobre los elementos. Se escriben en pares de nombre y valor: nombre="valor".

Atributos comunes:

  • id: identificador único para un elemento.
  • class: clasificación del elemento.
  • style: define los estilos CSS directamente en el elemento.

4️⃣ Etiquetas y Atributos Fundamentales de HTML

Reglas Generales ›
  • Todos los documentos HTML deben comenzar con una declaración de tipo de documento: <!DOCTYPE html>.
  • El documento HTML en sí comienza con <html> y termina con </html>. La parte visible del documento está entre <body> y </body>.
  • Las etiquetas HTML no son case sensitive: <H1> es igual a <h1>.
  • El estándar HTML5 no requiere etiquetas en minúscula, pero W3C recomienda las etiquetas en minúscula, y la define como obligatorio para documentos del tipo XHTML.
Encabezados y Párrafos ›

Los títulos HTML se definen con las etiquetas <h1> a <h6>. <h1> define el encabezado más importante o de mayor tamaño, y <h6> define el encabezado más pequeño.

La etiqueta párrafo se define con la etiqueta <p>.

Enlaces (Links) ›

Los links de HTML se definen con la etiqueta <a>. El destino del link se declara con el atributo href. Los atributos amplían la información del elemento.

De forma predeterminada, la página vinculada se muestra en la ventana del navegador actual. Para cambiar esto, se debe especificar otro destino con el atributo target:

Valor Descripción
_selfPredeterminado. Abre el documento en la misma ventana/pestaña en la que se hizo clic.
_blankAbre el documento en una nueva ventana o pestaña.
_parentAbre el documento en el marco principal.
_topAbre el documento en el cuerpo completo de la ventana.

Enlace a correo electrónico: se usa mailto: dentro del atributo href para crear un enlace que abra el programa de correo electrónico del usuario y le permita enviar un nuevo correo.

Imágenes ›

Las imágenes HTML se definen con la etiqueta <img>. El archivo de origen (src), el texto alternativo (alt), ancho (width) y alto (height) se declaran como atributos.

Etiquetas Vacías ›

Estas etiquetas no contienen contenido. <br> es un elemento vacío sin una etiqueta de fin. Puede cerrarse con la etiqueta de comienzo seguida de un slash: <br />.

Si se busca una validación estructurada o se usará para ser leído por analizadores XML, deberá cerrarlas. La etiqueta <br> sirve como salto de línea.

La Etiqueta <head> ›

Es un elemento contenedor de metadata acerca del documento HTML. Esta información no es visualizada. El elemento <head> está ubicado entre la etiqueta <html> y <body>.

El Atributo style ›

El atributo style se usa para agregar estilos a un elemento, como color, fuente, tamaño y más.

  • background-color: cambia el color del fondo.
  • color: color del texto.
  • font-family: fuente de los textos.
  • font-size: tamaño de la letra.
  • text-align: alinea el texto.
El Atributo lang ›

El atributo lang se incluye dentro de la etiqueta <html>, para declarar el idioma de la página web. Esto está destinado a ayudar a los motores de búsqueda y navegadores.

Los códigos de país también se pueden agregar al código de idioma: los dos primeros caracteres definen el idioma y los dos últimos definen el país (ej: lang="en-US" especifica inglés como idioma y Estados Unidos como país).

Reglas Horizontales y Texto Preformateado ›

<hr>: define un salto temático en una página HTML y, con mayor frecuencia, se muestra como una regla horizontal. Se utiliza para separar contenido (o definir un cambio) en una página HTML.

<pre>: define texto preformateado. El texto dentro de un elemento <pre> se muestra en una fuente de ancho fijo y conserva tanto los espacios como los saltos de línea.

Formato de Texto: <b> y <strong> ›

<b> define texto en negrita.

<strong> define el resaltado del texto (semánticamente indica importancia, más allá del estilo visual).

HTML5 APIs
Geolocation API ›

Definición: devuelve coordenadas (latitud y longitud) de la ubicación del usuario.

Objeto: navigator.geolocation

Métodos:

  • .getCurrentPosition(): recupera la posición actual del usuario.
  • .watchPosition(): verifica la posición del usuario regularmente para detectar cambios.
  • .clearWatch(): finaliza un watchPosition().
Drag and Drop API ›

Definición: permite arrastrar y soltar elementos en la página.

Propiedad: draggable="true" hace un elemento arrastrable.

Eventos:

  • ondragstart: se dispara cuando comienza el arrastre.
  • ondragover: especifica dónde se puede arrastrar.
  • ondrop: se dispara cuando se suelta el elemento.

Métodos:

  • event.preventDefault(): permite el drop (debe usarse en ondragover).
  • dataTransfer.setData(): establece el tipo y valor de los datos arrastrados.
  • dataTransfer.getData(): obtiene los datos arrastrados.
Canvas API ›

Definición: permite dibujar gráficos en una página web a través de JavaScript.

  • Es solo un contenedor para gráficos.
  • Debe usarse JavaScript para dibujar.
  • Tiene varios métodos para dibujar trayectorias, cajas, círculos, texto e imágenes.

Rectángulos:

  • rect(): crea un rectángulo.
  • fillRect(x, y, width, height): rectángulo lleno.
  • strokeRect(x, y, width, height): rectángulo sin llenar (solo bordes).
  • clearRect(x, y, width, height): limpia píxeles específicos.

Trayectorias/Líneas:

  • beginPath(): comienza una trayectoria.
  • moveTo(x, y): mueve el punto sin crear línea.
  • lineTo(x, y): crea línea hasta el punto.
  • closePath(): crea trayectoria de vuelta al inicio.
  • stroke(): dibuja la trayectoria.
  • fill(): rellena la trayectoria.

Círculos, Arcos y Curvas:

  • arc(x, y, radius, startAngle, endAngle): crea círculo/arco.
  • arcTo(x1, y1, x2, y2, radius): arco entre dos tangentes.
  • quadraticCurveTo(): curva Bézier cuadrática.
  • bezierCurveTo(): curva Bézier cúbica.

Otros métodos: clip() (recorta región de cualquier forma), isPointInPath() (verifica si un punto está en la trayectoria).

Web Storage ›

Definición: permite almacenar datos localmente en el navegador del usuario sin enviarlos al servidor.

Ventajas:

  • La información nunca se transfiere al servidor.
  • Puede almacenar hasta 5 MB.
  • Mejor que las cookies para datos locales.

Objetos de almacenamiento:

  • localStorage: almacena datos sin fecha de expiración (permanente).
  • sessionStorage: almacena datos para una sesión (se pierden al cerrar el navegador).

Métodos y atributos:

  • .getItem(key): devuelve el valor del elemento con clave key.
  • .setItem(key, value): almacena valor referenciado por clave.
  • .removeItem(key): elimina el par clave/valor.
  • .clear(): elimina todos los elementos.
  • .key(index): devuelve la clave en la posición index.
  • .length: número de elementos almacenados.
Web Workers ›

Definición: código JavaScript que corre en segundo plano sin afectar la performance de la página.

Características:

  • Permite procesos pesados sin bloquear la interfaz.
  • No afectan la usabilidad de la aplicación.
  • Se ejecutan en paralelo con el código principal.

Tipos:

  • Web Workers Dedicados: vinculados al script que los llamó.
  • SharedWorker: accesibles desde múltiples contextos (varias ventanas, iframes, otros workers).

Métodos:

  • new Worker("archivo.js"): crea un worker.
  • .postMessage(data): envía mensajes al worker.
  • .onmessage: recibe mensajes del worker.
  • .terminate(): detiene el worker.

7️⃣ CSS (Cascading Style Sheets) ⭐ Examen

CSS describe cómo se visualizan los elementos del HTML.
Tres Formas de Usar CSS ›
  • Inline: mediante el atributo style directamente en el elemento.
  • Internal: mediante un elemento <style> dentro del <head>.
  • External: mediante un archivo CSS externo enlazado con <link rel="stylesheet">.
CSS Fonts (Fuentes) ›
  • color: define el color del texto.
  • font-family: define la fuente a utilizar.
  • font-size: define el tamaño del texto.

RGBFormato rgb(rojo, verde, azul), valores de 0 a 255 para cada color.

Imágenes en HTML ›

Consideraciones:

  • Las imágenes mejoran el diseño y apariencia de una página web.
  • No se insertan técnicamente en la página; están vinculadas a la página.
  • La etiqueta <img> crea un espacio de espera para la imagen referenciada.

Características de <img>: etiqueta vacía (sin contenido, sin cierre), solo contiene atributos, puede colocarse en cualquier lugar de la página.

Atributos obligatorios:

AtributoDescripción
srcEspecifica la ruta/ubicación de la imagen.
altEspecifica texto alternativo si la imagen no carga.

Tamaño de imágenes: se puede definir con los atributos width y height (en píxeles, más directo) o con el atributo style (más flexible, admite porcentajes).

Tablas HTML ›

Sirven para organizar datos en filas y columnas de forma estructurada.

Etiquetas principales:

EtiquetaDescripción
<table>Define la tabla completa.
<tr>Define una fila (Table Row).
<th>Define encabezado de tabla (Table Header).
<td>Define datos de celda (Table Data).

Estilos por defecto: <th> se muestra en negrita y centrado; <td> se muestra en texto regular y alineado a la izquierda.

Propiedades CSS para tablas:

  • border: agrega bordes a la tabla.
  • border-collapse: elimina el espacio doble entre bordes de celdas.
  • padding: agrega espacio interno dentro de las celdas.
  • text-align: alinea el contenido de las celdas (left, center, right).
  • border-spacing: define el espacio entre las celdas.

Atributos para celdas:

  • colspan: hace que una celda abarque múltiples columnas.
  • rowspan: hace que una celda abarque múltiples filas.

Identificador único id: define de forma única una tabla para estilos o referencias específicas.

9️⃣ Bloques, Formularios y Eventos

Elementos a Nivel de Bloque vs Línea ›

Cada elemento HTML tiene un valor de visualización predeterminado: bloque o en línea.

Elementos a nivel de bloque: siempre comienzan en una nueva línea, ocupan todo el ancho disponible y se extienden hacia izquierda y derecha tanto como pueden.

<div>: contenedor a nivel de bloque usado para agrupar otros elementos HTML. Sin atributos obligatorios (atributos comunes: style, class, id). Se usa con CSS para diseñar bloques de contenido, como contenedor de secciones, aplicar estilos a grupos de elementos y organizar la estructura de la página.

<span>: contenedor en línea usado para marcar una parte de texto o documento. No comienza nueva línea. Sin atributos obligatorios. Se usa con CSS para diseñar partes específicas de texto.

Diferencia clave: <div> = bloque (nueva línea); <span> = en línea (mismo renglón).
El Elemento <form> ›

Define un formulario para recolectar datos que ingresa el usuario. Contiene varios tipos de elementos de entrada, envía datos a un servidor e interactúa con el usuario.

Elementos de formulario: text fields (campos de texto), checkboxes (casillas de verificación), radio buttons (botones de opción), submit buttons (botones enviar), entre otros.

El Elemento <input> ›

Campo de entrada que puede visualizarse de diferentes formas según el atributo type.

Atributos: type (define el tipo de entrada), name (nombre del campo), value (valor predeterminado), placeholder (texto de ayuda).

Tipos de <input>:

El Elemento <select> - Desplegable ›

Crea un menú desplegable para seleccionar una opción.

Estructura: <select> es el contenedor, <option> define cada opción. El atributo value es el valor a enviar, y selected marca la opción predeterminada.

Permite seleccionar MÚLTIPLES opciones. Varias pueden estar marcadas. value es lo que se envía y checked marca la predeterminada.

Eventos en HTML
Un evento es una acción del usuario que puede disparar código JavaScript.
Eventos de Documento e Imagen ›

Documento:

  • onload: ocurre cuando termina de cargar el documento.
  • onunload: ocurre cuando el usuario quita el documento de la ventana.

Imagen:

  • onabort: ocurre cuando la carga de imagen es abortada.
Eventos de Formulario ›
  • onselect: se selecciona contenido del elemento.
  • onchange: el elemento del formulario cambia su valor.
  • onfocus: el elemento recibe el foco (es seleccionado).
  • onblur: el elemento pierde el foco.
  • onreset: se resetea/limpia el formulario.
  • onsubmit: se envía el formulario.
Eventos de Teclado ›
  • onkeypress: tecla es presionada y soltada.
  • onkeydown: tecla es pulsada (presionada).
  • onkeyup: usuario libera la tecla.
Orden de ejecución: keydown → keypress → keyup
Eventos de Mouse ›
  • onmousemove: el mouse se mueve sobre el elemento.
  • onmouseout: el mouse sale del elemento.
  • onmouseover: el mouse entra al elemento desde afuera.
  • onclick: click del botón izquierdo del mouse.
  • ondblclick: doble click del mouse.
  • onmousedown: botón del mouse es presionado.
  • onmouseup: botón del mouse es soltado.
