1️⃣ CSS - Cascading Style Sheets ⭐ Examen

CSS (Cascading Style Sheets, "Hojas de Estilo en Cascada") es una tecnología utilizada para dotar de cualidades visuales y estéticas a una página web. Su concepto fundamental es aplicar estilos (colores, formas, márgenes, fuentes, etc.) a uno o varios documentos (generalmente HTML) de forma masiva y centralizada.
¿Por Qué se Llama "en Cascada"? ›
  • Aplicación de arriba a abajo: los estilos se aplican siguiendo un patrón jerárquico.
  • Herencia: los estilos se heredan de elementos padres a elementos hijos.
  • Resolución de ambigüedad: cuando hay conflictos entre estilos, se siguen normas específicas para resolverlos.
Elementos de Bloque vs En Línea ›
Bloque (Block)En Línea (Inline)
Comienzan en una nueva línea. Ocupan todo el ancho disponible. Ejemplos: <div>, <p>, <h1>, <section>, <header>, <footer>. Pueden comenzar en cualquier parte de una línea. Solo ocupan el ancho necesario. Ejemplos: <span>, <a>, <img>, <strong>, <em>.
Selectores CSS
¿Qué es un Selector CSS? ›

El selector CSS es el identificador en la hoja de estilos del elemento o elementos del DOM (Document Object Model) a los que se aplicarán los estilos.

El selector es el nexo de unión entre la hoja de estilos y los documentos HTML. Del buen manejo de selectores depende la eficiencia de la hoja de estilos, y permite aplicar estilos sin recurrir a otros lenguajes.
1. Selectores Simples ›

Son selectores básicos que seleccionan elementos directamente.

SelectorDescripción
De Elemento (Type)Selecciona por nombre de etiqueta (ej. todos los <p>).
De Clase (.clase)Selecciona elementos con una clase específica. Se puede usar en múltiples elementos.
De ID (#id)Selecciona un elemento único con un ID específico. Debe ser único en la página.
Universal (*)Selecciona todos los elementos.
2. Selectores Compuestos ›

Combinan selectores simples para mayor especificidad.

  • Selector de Atributo: selecciona elementos según sus atributos (ej. inputs con determinado type, enlaces con target específico, elementos con un atributo presente).
  • Selector Descendiente: selecciona elementos dentro de otros elementos (sin importar el nivel de anidamiento).
  • Selector Hijo (Child Combinator): selecciona elementos que son hijos directos de otro.
  • Selector Hermano Adyacente: selecciona un elemento que sigue inmediatamente a otro.
3. Selectores Complejos ›

Son combinaciones avanzadas de selectores, ya sea agrupando múltiples selectores separados por comas para aplicar el mismo estilo a varios elementos distintos, o encadenando selectores de tipo, clase y combinadores para apuntar a estructuras muy específicas del documento.

4. Pseudo-clases y Pseudo-elementos ›

Pseudo-clases: seleccionan elementos en estados específicos.

:link (no visitado), :visited (visitado), :hover (al pasar el mouse), :active (activo/presionado), :first-child (primer hijo), :last-child (último hijo), :focus (en foco).

Pseudo-elementos: seleccionan partes específicas de un elemento.

::first-letter (primera letra), ::first-line (primera línea), ::before (antes del contenido), ::after (después del contenido).

Tres Formas de Usar CSS
Inline, Internal y External ›
FormaCaracterísticas
Inline Los estilos se definen directamente en el elemento HTML usando el atributo style. Afecta solo al elemento específico, es menos eficiente para múltiples elementos y difícil de mantener.
Internal (Interno) Los estilos se definen en un elemento <style> dentro del <head>. Afecta a toda la página, pero solo funciona en esa página. Más fácil de mantener que inline.
External (Externo) Los estilos se definen en un archivo .css separado, vinculado al HTML. Mejor práctica para proyectos medianos/grandes: reutilizable en múltiples páginas, fácil de mantener y actualizar, reduce redundancia de código.
Ventajas de CSS Externo: un archivo CSS para múltiples HTML, los cambios se aplican en todo el sitio, mejor rendimiento (cache del navegador) y separación de responsabilidades.
Layout
¿Qué es un Layout? ›

La palabra "layout" se traduce como disposición, plan, diseño o esquema. En diseño web, el layout es la representación de un plano sobre el cual se va a dibujar la distribución de un espacio específico o determinado.

Importancia: sirve como base fundamental para una página web, es el producto inicial de un sitio web y define cómo se distribuyen los elementos dentro de la página.

Flujo de desarrollo: el diseñador crea el layout (esquema/boceto) → lo presenta al cliente para aprobación/modificación/rechazo → se desarrolla la página web basándose en el layout aprobado.

Tipos de Layouts Comunes ›
TipoEstructura
Una columnaHeader, Contenido y Footer apilados verticalmente.
Dos columnasHeader arriba, luego Sidebar junto a Contenido, y Footer abajo.
Tres columnasHeader arriba, luego una barra izquierda, el Contenido central y una barra derecha, y Footer abajo.

Elementos típicos de un layout:

  • Header: logotipo, navegación principal, información de la empresa.
  • Navigation: menú principal, enlaces a secciones.
  • Sidebar: widgets, enlaces secundarios, información adicional.
  • Main Content: artículos, información principal.
  • Footer: copyright, enlaces secundarios, información de contacto.
Propiedades CSS Importantes ›

Texto y fuentes: color, font-family, font-size, font-weight, text-align, line-height, text-decoration.

Box Model (modelo de caja): margin (espacio exterior), padding (espacio interior), border, width, height.

Colores y fondo: background-color, background-image, opacity.

Posicionamiento:

  • static: flujo normal (predeterminado).
  • relative: relativo a su posición normal.
  • absolute: relativo al contenedor padre.
  • fixed: fijo en la ventana.

Display: block, inline, inline-block (híbrido), flex, grid, none (oculto).

2️⃣ Introducción a JavaScript ⭐ Examen

JavaScript es un lenguaje de programación que se utiliza principalmente para aportar dinamismo a sitios y aplicaciones web. Funciona en complemento con HTML y CSS, permitiendo crear nuevas funcionalidades e interacciones avanzadas con los usuarios, e interactuar con otras aplicaciones o servicios de backend.
Lenguaje Interpretado ›

A diferencia de un lenguaje compilado, JavaScript es un lenguaje interpretado: se ejecuta por medio de un programa intérprete.

Los navegadores son los programas que se encargan de interpretar y ejecutar el código JavaScript. Cuando accedemos a una página, el navegador lee y ejecuta todos los archivos que conforman la aplicación (HTML, CSS, JS).

Sitios Web vs Aplicaciones Web ›
Sitios WebAplicaciones Web
Ofrecen principalmente información estática, sin modificación de contenido según las acciones del usuario. Hoy es poco común crear solo sitios estáticos. Permiten a los usuarios llevar a cabo diversas tareas. Son entornos interactivos y dinámicos, en continuo mantenimiento y actualización. Ejemplos: MercadoLibre, Youtube, Gmail, Facebook.

Características de las Aplicaciones Web:

  • Funcionalidades diversas: distintas tareas que usuarios y clientes pueden realizar.
  • Interactividad: animaciones y transiciones complejas.
  • Respuestas a eventos: capturar y enviar datos mediante formularios.
  • APIs y Backend: consumen servicios de backend y/o base de datos para cargar y administrar información.
Front-End y Back-End ›

Front-End (Cliente): parte de la aplicación que se ejecuta en el navegador, con la que interactúan los usuarios. Construido con HTML, CSS y JavaScript, integrando los tres lenguajes en un único producto. Tecnologías: HTML5, CSS3, JavaScript, frameworks (React, Angular, Vue), librerías y herramientas.

Back-End (Servidor): lado del servidor responsable de la gestión de datos. Aloja servicios y APIs, administra la información de la página. Tecnologías: Node.js, PHP, Laravel; bases de datos (MySQL, PostgreSQL); servidores (Apache, Nginx).

JavaScript facilita la comunicación e intercambio de información con APIs u otras aplicaciones mediante solicitudes HTTP.
Historia de JavaScript ›

1995: primera aparición pública de JavaScript, utilizado como herramienta del navegador Netscape Navigator. Objetivo original: permitir agregar programas a páginas web.

Evolución: ES1 (1997), ES2 (1998), ES3 (1999), ES5 (2009, actualizaciones significativas), ES6 (2015, mejoras importantes en sintaxis y funcionalidad).

Actualmente JavaScript se encuentra en la versión 11 y en desarrollo de la 12, con enfoque en las versiones ES6 y posteriores.

Diferencia con Java ›

Java: funciona a través de declaraciones que se establecen a partir de un sistema de clases en tiempo de compilación.

JavaScript: se basa en un sistema de tiempo de ejecución donde cada dato distinto puede representar valores booleanos, cadenas de texto (strings) o valores numéricos.

A pesar de los nombres similares, Java y JavaScript son lenguajes completamente diferentes.
Elementos Básicos de JavaScript ›
  • Datos: valores numéricos, cadenas, booleanos.
  • Variables: contenedores para almacenar datos.
  • Objetos: estructuras complejas con propiedades.
  • Arrays: colecciones de datos.
  • Funciones: bloques de código reutilizables.
Sintaxis: Reglas Básicas ›
  • Espacios en blanco y nuevas líneas: no se tienen en cuenta (igual que en HTML). El código se puede escribir en una o múltiples líneas.
  • Case sensitive: se distinguen mayúsculas de minúsculas (nombre ≠ Nombre).
  • Comentarios: de una línea con //, y de múltiples líneas entre /* y */.
Palabras Reservadas ›

Son palabras clave que se emplean para crear las instrucciones en JavaScript y que, por lo tanto, no pueden ser utilizadas de manera arbitraria como nombres de variables.

break, case, catch, continue, default, let, delete, do, else, finally, for, function, if, in, instanceof, new, return, switch, this, throw, try, typeof, var, void, while, with.

Variables ›

Una variable representa un área en la memoria que puede modificar su contenido durante la ejecución de un programa. En las variables se guardan diferentes tipos de datos que empleamos en la aplicación.

Declaración: declarar una variable significa su creación, usando las palabras reservadas var, let o const, seguidas del nombre de la variable.

Reglas para los nombres: evitar espacios y caracteres especiales; es case sensitive.

Valores y Tipos de Datos ›

Los dos tipos de datos más comunes para empezar son:

  • Number: un valor numérico que puede ser entero o decimal.
  • String: un valor compuesto por uno o más caracteres, definido entre comillas simples o dobles.
Asignación e Inicialización ›

Asignación: en una variable podemos asignar distintos tipos de valores mediante el operador de asignación, el símbolo =.

Inicializar variables: podemos declarar una variable y asignarle un valor inicial en el mismo proceso.

Asignación de múltiples variables: se puede declarar varias variables en una instrucción, comenzando con var/let/const y separando las variables con comas (también se puede expresar en múltiples líneas para mayor claridad).

let vs const ›

Las declaraciones con let y const ofrecen controles adicionales para las variables: en esencia, evitan la creación de dos variables con el mismo nombre.

let: puede recibir varias asignaciones a lo largo de la aplicación; su valor puede cambiar en diferentes momentos.

const: solo puede ser asignada una vez, en el momento de su declaración. Impide cualquier modificación de su valor posteriormente.

Operadores Aritméticos ›
  • Suma (+): suma dos números.
  • Resta (-): resta un número de otro.
  • Multiplicación (*): multiplica dos números.
  • División (/): divide un número entre otro.
  • Módulo (%): devuelve el resto de una división.
  • Incremento (++): aumenta el valor de una variable en uno.
  • Decremento (--): disminuye el valor de una variable en uno.
Operaciones con Números y con Strings ›

Con números: con variables de valores numéricos se pueden realizar operaciones matemáticas: sumas, restas, multiplicaciones, divisiones, etc.

Con strings (concatenación): con variables de tipo string se pueden concatenar los valores, es decir, combinarlos mediante el operador +.

Sumando números y letras: sumando dos números se obtiene el resultado numérico. Sin embargo, si se suma un número y una string, el resultado será un string. De igual manera, si se suma palabras con números, el resultado será una concatenación de texto.
Operadores de Comparación y Lógicos ›

Operadores de comparación: comparan valores y devuelven un resultado verdadero o falso.

  • Igual a (==): compara si dos valores son iguales.
  • Igual valor, igual tipo (===): compara si dos valores son iguales en valor y en tipo.
  • No igual (!=): compara si dos valores no son iguales.
  • Distinto valor o distinto tipo (!==): compara si dos valores son diferentes en valor o en tipo.
  • Mayor que (>): compara si un valor es mayor que otro.
  • Menor que (<): compara si un valor es menor que otro.
  • Mayor o igual a (>=): compara si un valor es mayor o igual que otro.
  • Menor o igual a (<=): compara si un valor es menor o igual que otro.

Operadores lógicos: se utilizan para determinar la lógica entre variables o valores. Son fundamentales para tomar decisiones en la programación y se utilizan comúnmente en estructuras de control como condicionales y bucles.

Sentencias: prompt(), console.log() y alert() ›

Una sentencia es un bloque de código que le dice al navegador que realice una acción específica. Las instrucciones se ejecutan una por una, en el orden que fueron escritas (flujo secuencial), y cada instrucción debe estar separada por punto y coma (;).

prompt(): abre un cuadro de diálogo que permite al usuario introducir información. Es posible agregar un mensaje que aparecerá encima del campo de texto. Devuelve una cadena (string) con lo que el usuario escribió.

console.log(): imprime en la consola del navegador el mensaje que se le proporciona como argumento. En Chrome se accede haciendo clic derecho → "Inspeccionar" → pestaña "Consola".

alert(): muestra una ventana emergente sobre la página con el mensaje pasado como parámetro. El valor que se muestra al usuario se denomina salida.

Visualización de datos: JavaScript puede mostrar datos en una ventana de alerta, en el documento HTML, o en un elemento HTML específico.

Funciones ›

Una función es un bloque de código que puede ser ejecutado cuando ocurre un evento (por ejemplo, cuando se presiona un botón). Se definen con la palabra clave function, seguida por un nombre y las instrucciones encerradas entre llaves { }. El nombre puede contener letras, números y guión bajo.

Las funciones pueden tener parámetros (nombres de variables separados por comas dentro de los paréntesis) que permiten pasarles valores.

Formas de ejecución: cuando ocurre un evento, cuando es llamada por una instrucción JavaScript, o de manera automática.

Function Return: cuando JavaScript encuentra una instrucción return, la función deja de ejecutarse. Las funciones a menudo devuelven un valor producto de las instrucciones que efectuaron, y ese valor lo usará la instrucción que la llamó.

Reutilización: las funciones se crean para reutilizar código, usando el mismo código con distintos argumentos para producir resultados diferentes. Una función puede usarse igual que las variables en fórmulas, asignaciones y cálculos.

Bloques de código: son instrucciones que se ejecutan juntas (por ejemplo, una función) y están encerradas entre llaves { }. Se utilizan en funciones, condicionales, bucles y otros constructos de control de flujo.

Objetos, Propiedades y Métodos ›

Un objeto es una variable especial que puede contener muchos valores. Los valores se escriben como pares nombre:valor, llamados propiedades. Múltiples propiedades se separan por comas.

Propiedades: atributos o características de un objeto (ej: un auto tiene nombre, modelo, color, peso).

Métodos: acciones o funciones que puede realizar un objeto (ej: un auto puede arrancar, frenar, apagar).

Manipulación del HTML y Ubicación del Código ›

JavaScript puede cambiar el contenido del HTML, los atributos, los estilos, y puede ocultar o mostrar elementos.

document.getElementById(): se utiliza para acceder a elementos del HTML por su id y modificarlos desde JavaScript.

innerHTML: propiedad usada para cambiar el contenido de un elemento HTML.

Ubicación del código: puede estar en las secciones <head> y <body> de un documento HTML, escrito entre <script> y </script>, o en un archivo externo con extensión .js, vinculado con la etiqueta <script> y el atributo src.

Herramientas Recomendadas ›

Editores de código: Visual Studio Code, Sublime Text, Atom.

Frameworks: Bootstrap 5, Bulma, Tailwind CSS.

Librerías: moment JS, Sweet Alert, Axios.

Servidores: Live Server (VS Code), XAMPP, WampServer.

Relación entre HTML, CSS y JavaScript ›

HTML: aporta, a través de sus etiquetas, la estructura básica del sitio web. Funciona en consonancia y en relación con CSS y JS.

CSS: se utiliza para darle el estilo que se quiere mostrar en el sitio web a través de la definición de formato y diseño de la presentación.

JavaScript: utiliza a ambos lenguajes (HTML y CSS) para controlar el comportamiento y funcionalidad de los elementos del sitio web.

Estos tres lenguajes trabajan juntos para crear una experiencia web completa e interactiva.
Algoritmo y Programa ›

Algoritmo: una serie de pasos o funciones organizados que son necesarios para llevar a cabo una operación o acción específica (ej: en una suma, el algoritmo consiste en tomar un valor, añadirlo a otro y obtener un resultado).

Programa: un conjunto de instrucciones interpretadas por una computadora. En JavaScript, cada instrucción debe estar separada por punto y coma (;).

3️⃣ DOM - Document Object Model

El DOM (Document Object Model) es una representación estructurada del documento HTML en forma de árbol. Cada elemento HTML en el documento se convierte en un nodo en este árbol. El DOM es una interfaz que permite a JavaScript interactuar con los elementos HTML del documento, accediendo a ellos, modificándolos, eliminándolos o creando nuevos elementos.
Nodos del DOM ›

El DOM está conformado por todas las etiquetas que posee el documento HTML. Para acceder a cada uno de estos nodos se utiliza el objeto document, que contiene toda la estructura de nodos del documento.

Tipos principales de nodos:

  • Nodo Documento: el nodo raíz que contiene todo el documento.
  • Nodo Elemento: cada etiqueta HTML del documento.
  • Nodo Atributo: los atributos de los elementos HTML.
  • Nodo Texto: el contenido de texto dentro de los elementos.
El objeto document es el punto de entrada principal para acceder y manipular el DOM. Representa el documento HTML cargado en el navegador y contiene todos los métodos necesarios para interactuar con sus nodos.
Métodos para Acceder a Elementos ›
MétodoDescripción
getElementById()Devuelve el elemento con el ID especificado (busca un elemento único por su atributo id).
getElementsByTagName()Devuelve una colección con todos los elementos con el nombre de etiqueta especificado.
getElementsByClassName()Devuelve una colección de todos los elementos que contienen la clase especificada.
getElementsByName()Devuelve una lista de elementos que tienen el atributo name especificado.
querySelector()Devuelve el primer elemento que coincide con el selector CSS especificado.
querySelectorAll()Devuelve una lista de todos los elementos que coinciden con el selector CSS especificado.
Clasificación: getElementById(), getElementsByTagName(), getElementsByClassName() y getElementsByName() son métodos más antiguos (aunque funcionan, se consideran menos modernos). querySelector() y querySelectorAll() son métodos modernos y recomendados: utilizan selectores CSS y son más flexibles y poderosos.
Métodos para Crear y Manipular Nodos ›

Para crear nodos:

  • createElement(): crea un nuevo nodo elemento con el nombre de etiqueta especificado.
  • createTextNode(): crea un nuevo nodo de texto con el contenido especificado.
  • createAttribute(): crea un nuevo nodo de atributo con el nombre especificado.

Para manipular nodos:

  • appendChild(): añade un nuevo nodo secundario al final del nodo especificado.
  • removeChild(): elimina el nodo secundario especificado del nodo padre.
  • replaceChild(): reemplaza un nodo secundario existente con uno nuevo.
  • insertBefore(): inserta un nuevo nodo secundario antes del nodo secundario especificado.

Para manipular atributos:

  • getAttribute(): devuelve el valor del atributo especificado de un elemento.
  • setAttribute(): establece o modifica el valor del atributo especificado.
Eventos y Listeners ›

Una particularidad importante del DOM es que se pueden añadir "listeners" (escuchadores) a diferentes elementos. Los listeners escuchan posibles eventos que pueden darse en distintas etiquetas o elementos.

Un evento es una acción que ocurre en un elemento, como hacer clic, escribir en un campo de formulario, cargar una página, mover el mouse, etc.

Métodos para gestionar eventos:

  • addEventListener(): registra un evento en un objeto específico, asociando una función a un evento particular. Cuando ocurre el evento, la función se ejecuta. Permite añadir múltiples listeners al mismo evento en un mismo elemento.
  • removeEventListener(): remueve del objeto un detector de evento previamente registrado con addEventListener.

Tipos de eventos comunes:

click, submit, change, mouseover, mouseout, keydown, keyup, load, focus, blur.

Flujo de eventos: Captura (el evento viaja desde el nodo raíz hasta el elemento destino) → Objetivo (el evento llega al elemento que fue disparado) → Burbuja (el evento viaja desde el elemento destino hacia el nodo raíz).
Manipulación de Contenido y Estilos ›

innerHTML: modifica el contenido HTML completo de un elemento, incluyendo cualquier elemento secundario.

textContent: modifica solo el contenido de texto de un elemento, sin interpretar etiquetas HTML.

Manipulación de estilos: accediendo a la propiedad style del elemento se pueden modificar estilos individuales, cambiando propiedades CSS como color, tamaño de fuente, fondo, entre otros.

Navegación por el Árbol del DOM ›
  • parentNode: accede al nodo padre de un elemento.
  • childNodes: accede a todos los nodos secundarios de un elemento.
  • firstChild: accede al primer nodo secundario.
  • lastChild: accede al último nodo secundario.
  • nextSibling: accede al nodo hermano siguiente.
  • previousSibling: accede al nodo hermano anterior.

4️⃣ Usabilidad e Interfaz de Usuario ⭐ Examen

Experiencia del Usuario (UX) ›

La experiencia del usuario no se trata del funcionamiento interno de un producto o servicio, sino de cómo funciona en el exterior: es cómo se siente interactuar con el producto. Es la interacción que una persona tiene cuando entra en contacto con un producto o servicio.

Aspectos clave: ¿Es difícil hacer cosas simples? ¿Es fácil de descifrar? ¿Cómo se siente al interactuar con el producto?

Cada producto utilizado por alguien crea una experiencia de usuario, ya sea un objeto físico, software, libro o cualquier interfaz.

Usabilidad ›

En términos generales, la usabilidad es la forma en que el diseño de un objeto facilita o no su uso. Se refiere a la capacidad de un software de ser atractivo, comprendido y finalmente usado por el usuario.

La usabilidad depende tanto del producto como del usuario: no puede verificarse solo desde la perspectiva del producto, sino que requiere considerar la interacción entre ambos.

Según la norma ISO 9241-111, la usabilidad es la medida en la cual un producto puede ser usado por determinados usuarios para conseguir objetivos específicos con efectividad, eficiencia y satisfacción en un contexto específico de uso.
Historia de la Usabilidad ›

Se inició alrededor de 1900, durante la Revolución Industrial, con la incorporación creciente de máquinas y herramientas: fue importante optimizar la facilidad de uso y la eficiencia de las máquinas para mejorar la producción de bienes y servicios. Esta es la raíz de la usabilidad moderna.

Años después se aplicó para mejorar la facilidad de uso de los primeros aparatos de telecomunicación como el telégrafo y el telegrama, y se empezó a aplicar en el campo de la ergonomía, donde las máquinas y herramientas debían coincidir con las características fisiológicas, anatómicas y psicológicas del trabajador.

A partir de 1990, la usabilidad crece de forma exponencial y comienzan a surgir conceptos como diseño de interacción y arquitectura de información.

Componentes de la Usabilidad ›
  • Facilidad de Aprendizaje (Learnability): ¿qué tan fácil resulta llevar a cabo tareas básicas la primera vez que se enfrenta al diseño?
  • Eficiencia: una vez aprendido, ¿cuánto tardan los usuarios en la realización de tareas?
  • Calidad de ser recordado (Memorability): después de no usarlo un tiempo, ¿cuánto tardan en volver a adquirir el conocimiento necesario para usarlo eficientemente?
  • Eficacia: ¿pueden los usuarios lograr sus objetivos?
  • Satisfacción: ¿cómo se sienten los usuarios al usar el producto?
Usabilidad vs Accesibilidad ›

La accesibilidad extiende el alcance del diseño para que sea efectivo para un número mayor de personas, mientras que la usabilidad se enfoca en facilitar el uso para los usuarios objetivo.

Una página web accesible es aquella que pone su usabilidad a disposición de absolutamente todos los visitantes, incluyendo personas con discapacidades. Sin accesibilidad, la usabilidad de una página web es inalcanzable para personas con discapacidad.

La accesibilidad se ve limitada por factores como discapacidades permanentes o temporales, nivel de conocimientos y experiencia, o limitaciones del software, hardware o entorno de uso.

Usabilidad y accesibilidad son resultado de un diseño centrado en los usuarios que conforman la audiencia del producto o servicio.
10 Reglas de Usabilidad Web de Jakob Nielsen ›
  • Visibilidad del estado del sistema: el sistema siempre debe mantener informados a los usuarios sobre lo que está sucediendo, con comentarios apropiados en un tiempo razonable.
  • Relación entre el sistema y el mundo real: usar frases y conceptos familiares al usuario, siguiendo las convenciones del mundo real, con información en orden natural y lógico.
  • Control y libertad del usuario: los usuarios eligen funciones por error y necesitarán una "salida de emergencia" claramente marcada para dejar el estado no deseado.
  • Consistencia y estándares: no debería haber dudas sobre si acciones, situaciones o palabras diferentes significan en realidad la misma cosa.
  • Prevención de errores: mejor que un buen diseño de mensajes de error es un diseño cuidadoso que prevenga la ocurrencia de problemas.
  • Reconocimiento antes que recuerdo: el usuario no debería tener que recordar información dada en una parte del proceso para seguir adelante; las instrucciones deben estar a la vista o ser fácilmente recuperables.
  • Flexibilidad y eficiencia de uso: la presencia de aceleradores permite una interacción más rápida a los usuarios expertos, permitiendo adaptar el sistema para usos frecuentes.
  • Estética y diseño minimalista: los diálogos no deben contener información irrelevante o poco usada; cada unidad extra compite con la información relevante y disminuye su visibilidad.
  • Ayudar a reconocer, diagnosticar y recuperarse de errores: los mensajes de error deben entregarse en lenguaje claro y simple, indicando el problema con precisión y sugiriendo una solución constructiva.
  • Ayuda y documentación: debería ser fácil de buscar, estar enfocada en las tareas del usuario, con una lista concreta de pasos y no ser demasiado extensa.
Métodos para Medir y Mejorar la Usabilidad ›
  • Inspección de usabilidad: expertos se centran en una lista de áreas de diseño que han demostrado ser problemáticas para los usuarios.
  • Inspección pluralista: usuarios, desarrolladores y gente de factores humanos se reúnen para discutir y evaluar paso a paso un escenario de tareas.
  • Inspección de consistencia: diseñadores expertos examinan productos o proyectos para garantizar la consistencia, verificando que las cosas estén hechas tal como fueron diseñadas.
  • Análisis de actividades: empleado en etapas preliminares de desarrollo para lograr el sentido de la situación; un investigador observa trabajar a un usuario en el campo.
  • Análisis de tareas: aprender acerca de los objetivos y formas de trabajar de los usuarios, y del entorno (físico, social, cultural, tecnológico).
  • Grupos de enfoque: debate donde un moderador dirige a un grupo de participantes mediante preguntas sobre un tema en particular.
  • Encuestas/Cuestionarios: costo reducido, no requieren equipo de prueba; los resultados reflejan la usabilidad percibida.
  • Métodos de prototipado: utilizados en las primeras etapas de desarrollo para validar y perfeccionar la usabilidad de un sistema.
  • Métodos de ensayo: generalmente grabados en video, proporcionan tiempo de finalización de tareas y permiten observar actitudes.
  • Prueba remota de usabilidad: explota el ambiente de los usuarios, transformándolo en un laboratorio.
  • Pensando en voz alta: conseguir que un usuario verbalice sus procesos de pensamiento a medida que desempeña una tarea.
  • Paseo cognitivo: usado para evaluar la facilidad de aprendizaje del sistema y comprender el proceso de pensamiento y toma de decisión del usuario.
  • Benchmarking: considera características clave como tiempo para hacer la tarea principal, tiempo para arreglar errores, tiempo para aprender y funcionalidad del sistema.
  • Personas: personajes ficticios creados para representar a los diferentes tipos de usuarios dentro de una demografía específica, utilizados en varias etapas del diseño.
Optimización de UX en 4 Pasos
Paso 1: Definir Personas ›

Una persona es una representación ficticia de los usuarios más comunes, basada en un conjunto compartido de tareas críticas comunes. Con una buena persona, se debe poder responder cualquier pregunta en el diseño o la investigación de UX para alinear las necesidades del usuario con las funciones o características del proyecto.

Dos tipos de personas: basadas en tareas críticas de usabilidad (tendrán tareas críticas especificadas) y de marketing o desarrollo de productos (no se incluyen en tareas críticas).

Las personas deben incluir: imagen (para personalizar y humanizar), tareas críticas (típicamente no más de 3), escenario (específico para la tarea), dispositivos (qué dispositivo suele utilizar), experiencia de dominio (si conoce el tema), y medio ambiente (contexto en el que interactúa).

Las personas guían decisiones de diseño, sirven como herramienta para controles de calidad preventivos y aseguran que todos estén de acuerdo con el objetivo final. El mayor error de los equipos de UX es no tener una persona o conjunto de personas cuidadosamente definida.
Paso 2: Análisis de Datos de UX de Comportamiento ›

Los datos de UX de comportamiento son de naturaleza cuantitativa. Se utilizan para identificar qué comportamientos ocurren o no en un sitio web, y qué cantidades y tipos de interacción están ocurriendo.

Los datos QUÉ junto con los datos de POR QUÉ (que provienen de pruebas de usabilidad) dan una visión completa de la actividad del sitio.

Incluyen datos de conversión de Google Analytics, CoreMetrics o herramientas relacionadas. Comprender qué atrae a los visitantes y si se convierten en clientes es importante para entender ciertos comportamientos de UX.

El índice de rebote (porcentaje de rebote de las páginas de contenido) puede indicar qué tan bien el contenido satisface las necesidades de los visitantes. Las visitas por navegador muestran qué navegadores son más utilizados para ver las páginas del sitio.

Paso 3: Pruebas de Usabilidad y UX ›

El objetivo es descubrir el POR QUÉ de los comportamientos: qué partes de las tareas críticas funcionan bien para los usuarios, qué partes no funcionan bien, qué confunde o les causa inquietud, y si se cumplen sus expectativas para la experiencia. Se hace observando a personas reales que coinciden con las personas definidas cuando intentan realizar sus tareas críticas.

Tipos de pruebas:

  • Prueba de usabilidad moderada: un moderador se conecta mediante comunicaciones bidireccionales en tiempo real con un sujeto de prueba. Permite hacer seguimiento, agregar datos críticos de POR QUÉ, probar con testers remotos, reducir costos y tiempo, y aprovechar intercepciones del sitio para probar con visitantes reales. Requiere herramientas de conexión en línea (Skype, Webex, GoToMeeting, JoinMe) y de captura/grabación (Camtasia, SnagIt, Adobe Captivate).
  • Prueba de usabilidad no moderada.
  • Test de 5 segundos: los visitantes tardan muy poco tiempo (en algunos casos una fracción de segundo, tan solo 50 milisegundos) para juzgar la calidad de un sitio web.
  • Test de click.
Paso 4: Analizar Resultados y Hacer Recomendaciones ›

Se buscan patrones que se alineen con comportamientos indeseables y se identifican oportunidades de optimización.

Tipos comunes de problemas de UX de comportamiento: errores de navegación o confusión donde el flujo del proceso no está alineado con el mapa mental del usuario, y otras cuestiones heurísticas según el sitio.

Solo porque los datos de investigación de comportamiento y UX parecen proporcionar recomendaciones de optimización, nunca se debe asumir que el análisis es correcto.
Signos de Problemas en la Definición de Persona ›
  • La persona no provino de observaciones reales del usuario.
  • La persona no tiene datos procesables.
  • La persona no tiene objetivos específicos.
  • La persona no proporciona métricas claras para el éxito.
  • La persona no coincide con el rango de edad.
  • La persona es demasiado genérica.
  • La persona es demasiado específica.
Ingeniería de Software y Usabilidad ›

La Ingeniería del Software se centró casi exclusivamente en atributos funcionales del sistema, lo que resultaba en usabilidad deficiente. Ahora está dirigida a un público cada vez más amplio, con usuarios cada vez menos expertos en el manejo de sistemas informáticos.

En los estándares ISO sobre calidad del software ya se distingue entre calidad inherente del software y calidad de uso.

La Ingeniería de Usabilidad es una aproximación al desarrollo de sistemas en la que se especifican niveles cuantitativos de usabilidad a priori (métricas), y el sistema se construye para alcanzar dichos niveles. Proporciona un modo práctico de asegurar que el software desarrollado alcanza un cierto nivel de usabilidad, basada en la evaluación mediante test de usabilidad con usuarios.
Interfaz de Usuario (UI)
¿Qué es la Interfaz de Usuario? ›

La interfaz de usuario es la vista que permite a un usuario interactuar de manera efectiva con un sistema. Una buena UI permite dar una buena UX, pero no lo es todo: es un instrumento.

Historia de la Interfaz de Usuario ›

Vannevar Bush propuso una herramienta llamada Memex: una mesa con teclado y palancas que permitiría la consulta de datos almacenados en microfilms proyectados en pantallas translúcidas, incluyendo la posibilidad de que el usuario realizara modificaciones (creador además de consultor).

Douglas Engelbart sentó las bases de la forma actual de interactuar con los computadores. A principios de los 60 desarrolló una combinación de computadora, teclado y pantalla con múltiples ventanas como base para la interacción hombre-máquina, además del principio Point & Click.

En 1973, XEROX PARC desarrolló una computadora con GUI (Graphical User Interface) basada en ventanas, iconos, menús y cursores (WIMP), probada en el Xerox Alto y comercializada en el Xerox Star (1981).

En 1979, MacOS siguió los trabajos de XEROX. El Apple Lisa (1984) fue el punto de partida para versiones más influenciadas por Xerox.

Microsoft creó el sistema operativo en modo texto DOS (MS-DOS). Su primera apuesta por el entorno gráfico fue Windows 1.0 (1985), seguido por Windows 2 (1987) y Windows 3 (1990).

Funciones Principales de la Interfaz ›

Puesta en marcha y apagado, control de las funciones manipulables del equipo, manipulación de archivos y directorios, herramientas de desarrollo de aplicaciones, comunicación con otros sistemas, información de estado, configuración de la propia interfaz y entorno, intercambio de datos entre aplicaciones, control de acceso, y sistema de ayuda interactivo.

Tipos de Interfaz de Usuario ›

Según la forma de interactuar del usuario: intérpretes de comando, interfaces gráficas, interfaces táctiles.

Según su construcción:

  • Interfaces de hardware: pulsadores, botones, teclas, reguladores, pantallas, diales, medidores, marcadores, instrumentos.
  • Interfaces de software: programas o parte de ellos que permiten expresar una orden a la computadora o visualizar su respuesta.

5️⃣ AJAX ⭐ Examen

AJAX no es una tecnología en sí misma, sino un conjunto de técnicas que permiten la comunicación asíncrona con el servidor, facilitando la interacción sin interrumpir la interfaz del usuario. Aunque se llama AJAX (Asynchronous JavaScript and XML), hoy en día se usa principalmente JSON en lugar de XML para el intercambio de datos.
Web Clásica vs Web Moderna ›
Antes de AJAX (Web Clásica)Con AJAX (Web Moderna)
Cada acción del usuario recargaba toda la página, resultando en una experiencia lenta y un consumo innecesario de recursos de red. El usuario tenía que esperar la recarga completa, interrumpiendo su flujo de trabajo. Permite la carga parcial de datos sin recargar la página, mejorando significativamente la experiencia del usuario y la eficiencia en el uso de la red, con interacciones más fluidas y rápidas.
Componentes de AJAX ›
  • JavaScript: para la lógica de cliente y la gestión de eventos.
  • XMLHttpRequest o Fetch API: para realizar solicitudes asincrónicas al servidor.
  • JSON o XML: para el formato de intercambio de datos.
  • DOM: para actualizar dinámicamente la página sin recargarla.
Beneficios y Cuándo Usar AJAX ›

Beneficios:

  • Experiencia de usuario fluida: interacción sin interrupciones causadas por recargas de página.
  • Menor carga en red: solo se transfieren los datos necesarios, no toda la página.
  • Interfaces interactivas: permite autocompletado, validaciones en tiempo real y actualizaciones inmediatas.
  • Separación frontend/backend: facilita una arquitectura más limpia donde ambos son independientes.

Cuándo usar AJAX:

  • Actualización de contenido sin recargar la página.
  • Interacciones en tiempo real con el servidor.
  • Validación de formularios sin recargar.
  • Carga de datos bajo demanda.
  • Aplicaciones web que requieren una experiencia similar a la de una aplicación de escritorio.
XMLHttpRequest ›

Es un objeto integrado en el navegador que permite comunicarse con el servidor mientras el usuario sigue usando la página. Es la tecnología que se usaba antes de la API Fetch.

Características: permite realizar peticiones HTTP asincrónicas (el navegador no se bloquea mientras se espera la respuesta), proporciona eventos para monitorear el estado de la solicitud, y devuelve la respuesta del servidor que puede ser parseada (convertida a objetos JavaScript).

Es ampliamente soportado en navegadores antiguos, pero su sintaxis es más compleja y menos intuitiva que Fetch, y requiere más código para manejar errores y estados.

readyState: ciclo de vida con diferentes estados.

ValorEstadoDescripción
0UNSENTLa solicitud ha sido creada pero aún no se ha ejecutado.
1OPENEDLa solicitud ha sido abierta.
2HEADERS_RECEIVEDSe han recibido los encabezados de la respuesta.
3LOADINGSe están recibiendo los datos de la respuesta.
4DONELa solicitud se ha completado (el valor más importante).

status: indica el resultado de la solicitud HTTP.

CódigoSignificado
200OK - la solicitud fue exitosa.
401Unauthorized - se requiere autenticación.
403Forbidden - el acceso al recurso está prohibido.
404Not Found - el recurso no fue encontrado.
500Internal Server Error - hubo un error en el servidor.

Se verifica que status === 200 para confirmar que la solicitud fue exitosa.

Flujo básico de una solicitud:

  1. Crear un objeto XMLHttpRequest.
  2. Especificar el método HTTP y la URL de la solicitud.
  3. Establecer un manejador de evento para cuando llegue la respuesta.
  4. Verificar que readyState === 4 (solicitud completada).
  5. Verificar que status === 200 (respuesta exitosa).
  6. Procesar los datos de la respuesta.
  7. Enviar la solicitud al servidor.
API Fetch ›

Fetch es una sintaxis moderna basada en Promises para hacer solicitudes HTTP. Es la alternativa moderna a XMLHttpRequest.

VentajasDesventajas
Sintaxis más limpia y legible. Usa Promises para un manejo mejor de errores. Compatible con async/await. Método integrado en navegadores modernos. Manejo de errores más intuitivo. No es soportada en navegadores muy antiguos (IE11 y anteriores). Requiere polyfills para navegadores antiguos.
Promises (Promesas) ›

Una Promise es un objeto especial en JavaScript que representa que algo va a pasar en el futuro: encapsula el resultado de una operación asincrónica.

Estados de una Promise:

  • Pendiente (pending): la operación aún no ha terminado, está en espera de un resultado.
  • Cumplida (fulfilled): la operación terminó exitosamente, la Promise se resolvió con un valor.
  • Rechazada (rejected): la operación terminó con un error, la Promise fue rechazada con un motivo.

Métodos para trabajar con Promises:

  • .then(): se ejecuta cuando la Promise se cumple, recibe el resultado como parámetro.
  • .catch(): se ejecuta cuando la Promise es rechazada, recibe el error como parámetro.
  • .finally(): se ejecuta siempre, sin importar si la Promise se cumplió o fue rechazada.
Comunicación Asincrónica ›

Permite que el navegador siga respondiendo a las acciones del usuario mientras se procesa una solicitud al servidor en segundo plano.

Ventajas: la interfaz no se congela mientras se espera la respuesta del servidor, mejor experiencia de usuario, y permite realizar múltiples solicitudes en paralelo.

Proceso de Parseo y Actualización del DOM ›

Parseo: proceso de convertir una cadena de texto (como JSON o XML) en un objeto de JavaScript que puede ser manipulado.

  • JSON.parse(): convierte una cadena JSON en un objeto JavaScript.
  • JSON.stringify(): convierte un objeto JavaScript en una cadena JSON.

El parseo es necesario porque los datos que vienen del servidor están en formato de texto y necesitan convertirse a estructuras de datos que JavaScript pueda entender y manipular.

Actualización del DOM con AJAX: una vez recibida la respuesta del servidor y parseados los datos, se actualiza la página utilizando métodos del DOM como appendChild() para añadir elementos dinámicamente, modificando innerHTML para cambiar contenido existente, o creando nuevos elementos con createElement() a los que se les asigna contenido.
